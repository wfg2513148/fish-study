# Codex 学习任务执行协议设计

## 目标

定义一套在 Codex Desktop 中使用的稳定协议，让家长上传照片后，Codex 能按固定步骤生成学习计划、打印版学习材料、错题复盘和 Obsidian 记录。协议服务于当前家庭学习流程，不构建独立 App，不要求孩子直接使用系统。

## 使用场景

### 场景 A：作业清单预习

家长上传老师布置的作业清单照片，并输入：

```text
帮我生成今日学习计划
```

Codex 需要先识别作业项，再定位相关知识点，最后输出：

- A4 学生打印版：做题前知识点、热身题、正式任务清单。
- 聊天里的家长参考：预计耗时、检查顺序、可能卡点。
- Obsidian 记录：当天学习计划、涉及知识点、后续复盘入口。

### 场景 B：错题贴纸复盘

家长上传已贴红黄蓝贴纸的错题卷照片，并输入：

```text
帮我生成错题知识点和测试题
```

Codex 需要识别题号、学科、题型、贴纸颜色和错因策略，输出：

- A4 学生打印版：知识点复盘、同类变式、小测。
- 聊天里的家长参考：答案、讲解顺序、二次贴纸建议。
- Obsidian 记录：错题归因、关联知识点、下次复测日期。

## 指令协议

### 指令 1：帮我生成今日学习计划

**输入：**

- 一张或多张作业清单照片。
- 可选补充：今天可用时间、孩子疲劳程度、优先学科。

**处理步骤：**

1. 识别作业清单。
2. 将作业项拆成结构化任务。
3. 从 Obsidian wiki 匹配年级、册别、学科、章节、知识点。
4. 对匹配不到的内容标记为 `待定位`，不得装作已确认。
5. 按学科和难度安排顺序。
6. 生成学生打印版和家长参考。
7. 写入 Obsidian 每日学习计划。

**输出：**

- `outputs/YYYY-MM-DD/today-study-plan.html`
- `/Users/kwang/Downloads/obsidian/fish-study/30-每日学习计划/YYYY-MM-DD.md`
- 聊天中的家长参考摘要。

### 指令 2：帮我生成错题知识点和测试题

**输入：**

- 一张或多张错题卷照片。
- 红黄蓝贴纸或标记：
  - 红色：不会
  - 黄色：马虎
  - 蓝色：时间不够

**处理步骤：**

1. 识别题号、题干、学生作答痕迹和贴纸颜色。
2. 根据颜色确定复盘策略。
3. 将题目匹配到 Obsidian 知识点。
4. 生成同类变式题和小测题。
5. 学生打印版不出现答案。
6. 家长参考在聊天中给答案和讲解要点。
7. 写入 Obsidian 错题归因记录。

**输出：**

- `outputs/YYYY-MM-DD/wrong-question-review.html`
- `/Users/kwang/Downloads/obsidian/fish-study/20-错题归因/YYYY-MM-DD.md`
- 被更新的知识点笔记中的“错题记录”段落。

### 指令 3：根据红黄蓝贴纸归因生成复习计划

**输入：**

- 一组近期错题记录或错题照片。

**处理步骤：**

1. 按知识点聚合红黄蓝贴纸数量。
2. 红色优先补概念和基础例题。
3. 黄色优先生成检查清单和易错提醒。
4. 蓝色优先生成限时训练和解题路径。
5. 对重复出现的红色知识点标记为 `高优先级复测`。

**输出：**

- 未来 3 到 7 天复习安排。
- 待复测知识点列表。
- Obsidian 复测记录。

## 结构化识别格式

Codex 在解释照片后，先形成内部结构，再生成面向家长和孩子的内容。

### 作业清单结构

```json
  "date": "YYYY-MM-DD",
  "items": [

      "book_or_source": "教材/练习册/试卷",
      "page": "页码或空",
      "question_range": "题号范围或空",
      "deadline": "截止时间或空",
```

## 知识点匹配规则

匹配顺序固定为：

1. 明确学科、年级、册别。
2. 匹配教材章节标题或课件标题。
3. 匹配知识点笔记标题。
4. 匹配笔记摘要、常见题型、易错点。
5. 匹配不到时标记 `待定位`。

置信度规则：

- `high`：题目或作业文字直接出现章节、课时、知识点名称。
- `medium`：可通过关键词和题型稳定推断。
- `low`：只有模糊题意，必须提示家长确认。

如果同一任务匹配到多个知识点，输出最多 3 个，并说明排序原因。不得把 `low` 置信度知识点写成确定结论。

## 打印输出规范

### 学生版

学生版用于打印，必须满足：

- 不出现答案。
- 每页 A4。
- 每个学科先给知识点，再给练习。
- 每次总题量控制在孩子能完成的范围内。
- 难度从基础到变式，不直接上难题。
- 页面底部保留自我检查区。

### 家长参考

家长参考只在聊天中输出，不单独打印，必须满足：

- 给答案或参考解法。
- 给检查顺序。
- 给预计用时。
- 标出需要家长确认的低置信度内容。

## 错题归因策略

| 颜色 | 归因 | 输出策略 |
|---|---|---|
| 红色 | 不会 | 先补知识点，再做基础变式，再做 1 道迁移题 |
| 黄色 | 马虎 | 生成审题、单位、符号、计算、书写检查清单 |
| 蓝色 | 时间不够 | 生成限时训练、步骤模板和优先级判断 |

升级规则：

- 同一知识点 7 天内出现 2 次红色：标记为 `高优先级复测`。
- 同一知识点连续 2 次黄色：生成专门检查清单。
- 同一题型连续 2 次蓝色：生成限时专项。


内容：

### 错题归因记录

路径：

```text
/Users/kwang/Downloads/obsidian/fish-study/20-错题归因/YYYY-MM-DD.md
```

内容：

- 错题列表。

### 知识点笔记更新

每个被命中的知识点笔记追加：

- 最近错题日期。
- 错因颜色。
- 复测状态。
- 关联打印材料。

## 质量自检

Codex 每次输出前必须自检：

1. 学生版是否不含答案。
2. 是否至少引用一个 Obsidian 知识点，或明确列出 `待定位`。
3. 是否把低置信度内容标给家长确认。
4. 题量是否适合当天时间。
5. 打印材料路径是否明确。
6. 错题归因是否使用红黄蓝规则。
7. 是否写入或计划写入 Obsidian 记录。

如果自检失败，Codex 不应直接给最终材料，而应先说明缺少什么信息。

## 错误处理

- 照片模糊：要求重新上传或让家长补充原文。
- 学科无法判断：列出候选学科，请家长确认。
- 知识点未命中：标为 `待定位`，不得编造成已命中。
- 题号不完整：生成材料时保留缺口，不扩写不存在的题号。
- 贴纸颜色无法判断：要求家长补充颜色，不擅自归因。

## 验收标准

协议设计完成后，后续实现必须满足：

- 三条指令都有固定输入、处理步骤、输出路径。
- 作业清单和错题都有结构化格式。
- 学生版永远不泄露答案。
- 家长参考和学生打印版分离。
- 所有低置信度知识点必须显式标注。
- Obsidian 中能追踪每日计划、错题归因和知识点复测。
- `待定位` 不阻塞输出，但必须进入待确认清单。

## 非目标

- 不构建独立前端 App。
- 不让孩子直接操作 Codex。
- 不自动联网查答案。
- 不提交原始教材、课件、照片或试卷到 GitHub。
- 不把低置信度推断写成确定知识点。
